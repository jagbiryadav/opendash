The OpenDash CLI by default starts the [TUI](/docs/tui) when run without any arguments.

```bash
opendash
```

But it also accepts commands as documented on this page. This allows you to interact with OpenDash programmatically.

```bash
opendash run "Explain how closures work in JavaScript"
```

---

### tui

Start the OpenDash terminal user interface.

```bash
opendash [project]
```

#### Flags

| Flag         | Short | Description                                                             |
| ------------ | ----- | ----------------------------------------------------------------------- |
| `--continue` | `-c`  | Continue the last session                                               |
| `--session`  | `-s`  | Session ID to continue                                                  |
| `--fork`     |       | Fork the session when continuing (use with `--continue` or `--session`) |
| `--prompt`   |       | Prompt to use                                                           |
| `--model`    | `-m`  | Model to use in the form of provider/model                              |
| `--agent`    |       | Agent to use                                                            |
| `--port`     |       | Port to listen on                                                       |
| `--hostname` |       | Hostname to listen on                                                   |

## Commands

The OpenDash CLI also has the following commands.

---

### agent

Manage agents for OpenDash.

```bash
opendash agent [command]
```

---

### attach

Attach a terminal to an already running OpenDash backend server started via `serve` or `web` commands.

```bash
opendash attach [url]
```

This allows using the TUI with a remote OpenDash backend. For example:

```bash
# Start the backend server for web/mobile access
opendash web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
opendash attach http://10.20.30.40:4096
```

#### Flags

| Flag        | Short | Description                       |
| ----------- | ----- | --------------------------------- |
| `--dir`     |       | Working directory to start TUI in |
| `--session` | `-s`  | Session ID to continue            |

---

#### create

Create a new agent with custom configuration.

```bash
opendash agent create
```

This command will guide you through creating a new agent with a custom system prompt and tool configuration.

---

#### list

List all available agents.

```bash
opendash agent list
```

### auth

Command to manage credentials and login for providers.

```bash
opendash auth [command]
```

---

#### login

OpenDash is powered by the provider list at [Models.dev](https://models.dev), so you can use `opendash auth login` to configure API keys for any provider you'd like to use. This is stored in `~/.local/share/opendash/auth.json`.

```bash
opendash auth login
```

When OpenDash starts up it loads the providers from the credentials file. And if there are any keys defined in your environments or a `.env` file in your project.

---

#### list

Lists all the authenticated providers as stored in the credentials file.

```bash
opendash auth list
```

Or the short version.

```bash
opendash auth ls
```

---

#### logout

Logs you out of a provider by clearing it from the credentials file.

```bash
opendash auth logout
```

### github

Manage the GitHub agent for repository automation.

```bash
opendash github [command]
```

---

#### install

Install the GitHub agent in your repository.

```bash
opendash github install
```

This sets up the necessary GitHub Actions workflow and guides you through the configuration process. [Learn more](/docs/github).

---

#### run

Run the GitHub agent. This is typically used in GitHub Actions.

```bash
opendash github run
```

##### Flags

| Flag      | Description                            |
| --------- | -------------------------------------- |
| `--event` | GitHub mock event to run the agent for |
| `--token` | GitHub personal access token           |

### mcp

Manage Model Context Protocol servers.

```bash
opendash mcp [command]
```

---

#### add

Add an MCP server to your configuration.

```bash
opendash mcp add
```

This command will guide you through adding either a local or remote MCP server.

---

#### list

List all configured MCP servers and their connection status.

```bash
opendash mcp list
```

Or use the short version.

```bash
opendash mcp ls
```

---

#### auth

Authenticate with an OAuth-enabled MCP server.

```bash
opendash mcp auth [name]
```

If you don't provide a server name, you'll be prompted to select from available OAuth-capable servers.

You can also list OAuth-capable servers and their authentication status.

```bash
opendash mcp auth list
```

Or use the short version.

```bash
opendash mcp auth ls
```

---

#### logout

Remove OAuth credentials for an MCP server.

```bash
opendash mcp logout [name]
```

---

#### debug

Debug OAuth connection issues for an MCP server.

```bash
opendash mcp debug <name>
```

### models

List all available models from configured providers.

```bash
opendash models [provider]
```

This command displays all models available across your configured providers in the format `provider/model`.

This is useful for figuring out the exact model name to use in [your config](/docs/config/).

You can optionally pass a provider ID to filter models by that provider.

```bash
opendash models anthropic
```

#### Flags

| Flag        | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `--refresh` | Refresh the models cache from models.dev                     |
| `--verbose` | Use more verbose model output (includes metadata like costs) |

Use the `--refresh` flag to update the cached model list. This is useful when new models have been added to a provider and you want to see them in OpenDash.

```bash
opendash models --refresh
```

### run

Run opendash in non-interactive mode by passing a prompt directly.

```bash
opendash run [message..]
```

This is useful for scripting, automation, or when you want a quick answer without launching the full TUI. For example.

```bash "opendash run"
opendash run Explain the use of context in Go
```

You can also attach to a running `opendash serve` instance to avoid MCP server cold boot times on every run:

```bash
# Start a headless server in one terminal
opendash serve

# In another terminal, run commands that attach to it
opendash run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flags

| Flag                             | Short | Description                                                             |
| -------------------------------- | ----- | ----------------------------------------------------------------------- |
| `--command`                      |       | The command to run, use message for args                                |
| `--continue`                     | `-c`  | Continue the last session                                               |
| `--session`                      | `-s`  | Session ID to continue                                                  |
| `--fork`                         |       | Fork the session when continuing (use with `--continue` or `--session`) |
| `--share`                        |       | Share the session                                                       |
| `--model`                        | `-m`  | Model to use in the form of provider/model                              |
| `--agent`                        |       | Agent to use                                                            |
| `--file`                         | `-f`  | File(s) to attach to message                                            |
| `--format`                       |       | Format: default (formatted) or json (raw JSON events)                   |
| `--title`                        |       | Title for the session (uses truncated prompt if no value provided)      |
| `--attach`                       |       | Attach to a running opendash server (e.g., http://localhost:4096)       |
| `--port`                         |       | Port for the local server (defaults to random port)                     |
| `--dangerously-skip-permissions` |       | Auto-approve permissions that are not explicitly denied (dangerous!)    |

### serve

Start a headless OpenDash server for API access. Check out the [server docs](/docs/server) for the full HTTP interface.

```bash
opendash serve
```

This starts an HTTP server that provides API access to opendash functionality without the TUI interface. Set `OPENCODE_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `opendash`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

opendash session list

##### Flags

### stats

Show token usage and cost statistics for your OpenDash sessions.

```bash
opendash stats
```

#### Flags

| Flag        | Description                                                                 |
| ----------- | --------------------------------------------------------------------------- |
| `--days`    | Show stats for the last N days (all time)                                   |
| `--tools`   | Number of tools to show (all)                                               |
| `--models`  | Show model usage breakdown (hidden by default). Pass a number to show top N |
| `--project` | Filter by project (all projects, empty string: current project)             |

---

### export

Export session data as JSON.

```bash
opendash export [sessionID]
```

If you don't provide a session ID, you'll be prompted to select from available sessions.

---

### import

Import session data from a JSON file or OpenDash share URL.

```bash
opendash import <file>
```

You can import from a local file or an OpenDash share URL.

```bash
opendash import session.json
opendash import https://opncd.ai/s/abc123
```

### web

Start a headless OpenDash server with a web interface.

```bash
opendash web
```

This starts an HTTP server and opens a web browser to access OpenDash through a web interface. Set `OPENCODE_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `opendash`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

### acp

Start an ACP (Agent Client Protocol) server.

```bash
opendash acp
```

This command starts an ACP server that communicates via stdin/stdout using nd-JSON.

#### Flags

| Flag         | Description           |
| ------------ | --------------------- |
| `--cwd`      | Working directory     |
| `--port`     | Port to listen on     |
| `--hostname` | Hostname to listen on |

---

### uninstall

Uninstall OpenDash and remove all related files.

```bash
opendash uninstall
```

#### Flags

| Flag            | Short | Description                                 |
| --------------- | ----- | ------------------------------------------- |
| `--keep-config` | `-c`  | Keep configuration files                    |
| `--keep-data`   | `-d`  | Keep session data and snapshots             |
| `--dry-run`     |       | Show what would be removed without removing |
| `--force`       | `-f`  | Skip confirmation prompts                   |

---

### upgrade

Updates opendash to the latest version or a specific version.

```bash
opendash upgrade [target]
```

To upgrade to the latest version.

```bash
opendash upgrade
```

To upgrade to a specific version.

```bash
opendash upgrade v0.1.48
```

#### Flags

| Flag       | Short | Description                                                       |
| ---------- | ----- | ----------------------------------------------------------------- |
| `--method` | `-m`  | The installation method that was used; curl, npm, pnpm, bun, brew |

## Environment variables

OpenDash can be configured using environment variables.

| Variable                              | Type    | Description                                       |
| ------------------------------------- | ------- | ------------------------------------------------- |
| `OPENCODE_AUTO_SHARE`                 | boolean | Automatically share sessions                      |
| `OPENCODE_GIT_BASH_PATH`              | string  | Path to Git Bash executable on Windows            |
| `OPENCODE_CONFIG`                     | string  | Path to config file                               |
| `OPENCODE_TUI_CONFIG`                 | string  | Path to TUI config file                           |
| `OPENCODE_CONFIG_DIR`                 | string  | Path to config directory                          |
| `OPENCODE_HOME`                       | string  | Absolute path to single profile root (holds `config/`, `data/`, `state/`, `cache/` subdirs); overrides all four XDG base dirs when set |
| `OPENCODE_CONFIG_CONTENT`             | string  | Inline json config content                        |
| `OPENCODE_DISABLE_AUTOUPDATE`         | boolean | Disable automatic update checks                   |
| `OPENCODE_DISABLE_PRUNE`              | boolean | Disable pruning of old data                       |
| `OPENCODE_DISABLE_TERMINAL_TITLE`     | boolean | Disable automatic terminal title updates          |
| `OPENCODE_PERMISSION`                 | string  | Inlined json permissions config                   |
| `OPENCODE_DISABLE_DEFAULT_PLUGINS`    | boolean | Disable default plugins                           |
| `OPENCODE_DISABLE_LSP_DOWNLOAD`       | boolean | Disable automatic LSP server downloads            |
| `OPENCODE_ENABLE_EXPERIMENTAL_MODELS` | boolean | Enable experimental models                        |
| `OPENCODE_DISABLE_AUTOCOMPACT`        | boolean | Disable automatic context compaction              |
| `OPENCODE_DISABLE_CLAUDE_CODE`        | boolean | Disable reading from `.claude` (prompt + skills)  |
| `OPENCODE_DISABLE_CLAUDE_CODE_PROMPT` | boolean | Disable reading `~/.claude/CLAUDE.md`             |
| `OPENCODE_DISABLE_CLAUDE_CODE_SKILLS` | boolean | Disable loading `.claude/skills`                  |
| `OPENCODE_DISABLE_MODELS_FETCH`       | boolean | Disable fetching models from remote sources       |
| `OPENCODE_DISABLE_MOUSE`              | boolean | Disable mouse capture in the TUI                  |
| `OPENCODE_FAKE_VCS`                   | string  | Fake VCS provider for testing purposes            |
| `OPENCODE_CLIENT`                     | string  | Client identifier (defaults to `cli`)             |
| `OPENCODE_ENABLE_EXA`                 | boolean | Enable Exa web search tools                       |
| `OPENCODE_SERVER_PASSWORD`            | string  | Enable basic auth for `serve`/`web`               |
| `OPENCODE_SERVER_USERNAME`            | string  | Override basic auth username (default `opendash`) |
| `OPENCODE_MODELS_URL`                 | string  | Custom URL for fetching models configuration      |
