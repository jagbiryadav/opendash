# Report Writing Prompts

## Executive Summary Generation
```
Generate an executive summary for the following analysis:

Analysis: {analysis_summary}
Audience: {audience}
Key Findings: {findings}
Recommendations: {recommendations}

Requirements:
1. Bottom Line Up Front (BLUF): One sentence answer
2. Key Findings: 3-5 most important insights
3. Recommendations: Prioritized actions
4. Business Impact: Quantified benefits
5. Confidence Level: High/Medium/Low

Tone: Direct, action-oriented, jargon-free
Length: 1-2 paragraphs
```

## Technical Report Generation
```
Generate a technical report for the following analysis:

Analysis: {analysis_summary}
Methodology: {methodology}
Results: {results}
Visualizations: {visualizations}

Structure:
1. Executive Summary
2. Methodology
3. Data Description
4. Exploratory Analysis
5. Statistical Analysis
6. Model Results (if applicable)
7. Interpretation
8. Limitations
9. Recommendations
10. Appendix

Requirements:
- Technical depth appropriate for analysts
- Statistical methods explained
- Code snippets where relevant
- All visualizations labeled and interpreted
```

## Dashboard Specification
```
Create a dashboard specification for the following use case:

Use Case: {use_case}
Audience: {audience}
Key Metrics: {metrics}
Update Frequency: {frequency}

Provide:
1. Layout: Visual arrangement of components
2. KPI Cards: Key metrics with trends
3. Charts: Visualizations with specifications
4. Filters: Interactive elements
5. Alerts: Conditions and thresholds
6. Mobile Considerations: Responsive design

Format: Structured specification document
```

## Presentation Outline
```
Create a presentation outline for the following analysis:

Analysis: {analysis_summary}
Audience: {audience}
Time Limit: {duration}
Objective: {objective}

Provide:
1. Title Slide: Clear, compelling title
2. Agenda: What will be covered
3. Context: Why this matters
4. Key Findings: 3-5 main points
5. Evidence: Data and visualizations
6. Recommendations: Actionable next steps
7. Q&A Preparation: Anticipated questions

Format: Slide-by-slide outline with speaker notes
```

## Stakeholder Communication
```
Draft a stakeholder communication for the following:

Analysis: {analysis_summary}
Stakeholder: {stakeholder}
Purpose: {purpose}
Urgency: {urgency}

Provide:
1. Subject Line: Clear and compelling
2. Opening: Why this matters to them
3. Key Points: 3-5 most relevant findings
4. Implications: What this means for them
5. Ask: What you need from them
6. Next Steps: What happens next

Tone: Professional, concise, action-oriented
Length: 1-2 pages
```

## Data Storytelling
```
Create a data story for the following:

Analysis: {analysis_summary}
Audience: {audience}
Key Message: {message}
Emotional Goal: {goal}

Provide:
1. Hook: Attention-grabbing opening
2. Context: Why this matters
3. Conflict: The problem or challenge
4. Resolution: What the data reveals
5. Call to Action: What to do next

Techniques:
- Use metaphors and analogies
- Create narrative arc
- Build emotional connection
- Make abstract concrete
```

## Limitations and Caveats
```
Document limitations for the following analysis:

Analysis: {analysis_summary}
Data: {data_description}
Methodology: {methodology}

Provide:
1. Data Limitations: Quality, completeness, bias
2. Methodological Limitations: Assumptions, constraints
3. Interpretation Limitations: Causation vs correlation
4. Scope Limitations: What was not covered
5. Uncertainty: Confidence levels and ranges
6. Recommendations: How to address limitations

Format: Clear, honest, constructive
```

## Follow-up Actions
```
Define follow-up actions for the following analysis:

Analysis: {analysis_summary}
Recommendations: {recommendations}
Timeline: {timeline}

Provide:
1. Immediate Actions (0-30 days)
2. Short-term Actions (30-90 days)
3. Long-term Actions (90+ days)
4. Success Metrics: How to measure progress
5. Ownership: Who is responsible
6. Resources: What is needed

Format: Actionable checklist with owners and deadlines
```

## Lessons Learned
```
Document lessons learned from the following analysis:

Analysis: {analysis_summary}
Challenges: {challenges}
Successes: {successes}

Provide:
1. What Worked Well: Successful approaches
2. What Could Improve: Areas for enhancement
3. Surprises: Unexpected findings or challenges
4. Recommendations: For future analyses
5. Knowledge Gaps: What we learned we need to know

Format: Constructive, forward-looking
```

## Quality Review
```
Review the following report for quality:

Report: {report_content}
Audience: {audience}
Purpose: {purpose}

Check:
1. Accuracy: Are all facts and figures correct?
2. Clarity: Is the writing clear and concise?
3. Completeness: Are all sections present?
4. Consistency: Is formatting consistent?
5. Impact: Does it achieve its purpose?
6. Actionability: Are recommendations clear?

Provide:
- Overall Assessment: Pass/Revise
- Strengths: What works well
- Issues: What needs fixing
- Recommendations: Specific improvements
```
