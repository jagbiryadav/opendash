# Report Writing Playbook

## Report Development Process

### Phase 1: Planning (Day 1)
**Objective**: Define report scope and audience

**Activities**:
1. Identify audience and their needs
2. Define report objectives
3. Outline key messages
4. Plan visualizations

**Outputs**:
- Report outline
- Audience analysis
- Key messages
- Visualization plan

### Phase 2: Content Development (Days 2-4)
**Objective**: Create report content

**Activities**:
1. Write executive summary first
2. Develop detailed sections
3. Create visualizations
4. Develop recommendations

**Outputs**:
- Executive summary
- Detailed sections
- Visualizations
- Recommendations

### Phase 3: Review and Refinement (Days 5-6)
**Objective**: Polish report for quality

**Activities**:
1. Technical review
2. Business review
3. Visual design review
4. Final edits

**Outputs**:
- Reviewed report
- Final edits
- Design improvements
- Quality assurance

### Phase 4: Delivery (Day 7)
**Objective**: Deliver report to stakeholders

**Activities**:
1. Final formatting
2. Distribution
3. Presentation preparation
4. Follow-up planning

**Outputs**:
- Final report
- Distribution list
- Presentation
- Follow-up plan

## Report Types

### Executive Summary (1-2 pages)
**Audience**: C-suite, senior leaders
**Focus**: Key findings and recommendations
**Tone**: Direct, action-oriented
**Structure**:
1. Bottom line up front (BLUF)
2. Key findings (3-5)
3. Recommendations (3-5)
4. Next steps

### Manager Report (3-5 pages)
**Audience**: VPs, directors, managers
**Focus**: Business context and implications
**Tone**: Strategic, evidence-based
**Structure**:
1. Executive summary
2. Business context
3. Key findings with evidence
4. Recommendations with implementation steps
5. Risks and mitigation

### Technical Report (10-20 pages)
**Audience**: Analysts, data scientists, technical leads
**Focus**: Methodology and detailed results
**Tone**: Technical, precise
**Structure**:
1. Executive summary
2. Methodology
3. Detailed results
4. Statistical analysis
5. Model details
6. Appendix

### Dashboard Report
**Audience**: Operational teams
**Focus**: Real-time metrics and trends
**Tone**: Clear, actionable
**Structure**:
1. KPI summary
2. Trend analysis
3. Segment breakdown
4. Alerts and actions

## Writing Best Practices

### Structure
1. **Inverted pyramid**: Most important information first
2. **Chunking**: Break content into digestible sections
3. **Signposting**: Use headers and transitions
4. **White space**: Let content breathe

### Language
1. **Active voice**: "We found..." not "It was found..."
2. **Simple sentences**: One idea per sentence
3. **Concrete language**: Specific numbers and examples
4. **Jargon-free**: Avoid technical terms for non-technical audience

### Visualizations
1. **One insight per chart**: Each chart tells one story
2. **Clear titles**: State the insight, not the topic
3. **Annotations**: Highlight key data points
4. **Consistent colors**: Same color = same meaning

## Quality Checklist

### Content
- [ ] Executive summary answers the question
- [ ] Key findings are quantified
- [ ] Recommendations are actionable
- [ ] Limitations are acknowledged
- [ ] Evidence supports all claims

### Structure
- [ ] Logical flow
- [ ] Clear section headers
- [ ] Smooth transitions
- [ ] Appropriate length for audience

### Visual Design
- [ ] Charts are clear and labeled
- [ ] Colors are consistent and accessible
- [ ] Layout is clean and professional
- [ ] Formatting is consistent

### Accuracy
- [ ] All numbers verified
- [ ] Sources cited
- [ ] Calculations double-checked
- [ ] No typos or grammatical errors

### Business Value
- [ ] Addresses the business question
- [ ] Recommendations are prioritized
- [ ] Impact is quantified
- [ ] Next steps are clear
