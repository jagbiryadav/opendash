# Analysis Quality Checklist

## Pre-Analysis

### Problem Definition
- [ ] Business problem clearly articulated
- [ ] Success criteria defined
- [ ] Stakeholders identified
- [ ] Timeline established
- [ ] Constraints documented

### Data Assessment
- [ ] Data sources identified
- [ ] Data access secured
- [ ] Data quality assessed
- [ ] Missing values documented
- [ ] Data limitations acknowledged

## During Analysis

### Exploratory Analysis
- [ ] All variables examined
- [ ] Distributions analyzed
- [ ] Relationships identified
- [ ] Outliers detected and assessed
- [ ] Temporal patterns analyzed (if applicable)

### Statistical Rigor
- [ ] Appropriate tests selected
- [ ] Assumptions checked
- [ ] Effect sizes reported
- [ ] Confidence intervals provided
- [ ] Multiple comparisons corrected (if applicable)

### Model Quality
- [ ] Baseline established
- [ ] Cross-validation performed
- [ ] No data leakage
- [ ] Performance on holdout set reported
- [ ] Model interpreted (SHAP/LIME)

## Post-Analysis

### Validation
- [ ] Findings validated with domain experts
- [ ] Alternative explanations considered
- [ ] Robustness checks performed
- [ ] Sensitivity analysis completed
- [ ] Limitations documented

### Business Impact
- [ ] Recommendations quantified
- [ ] Implementation feasibility assessed
- [ ] Risk assessment completed
- [ ] ROI estimated
- [ ] Next steps defined

### Documentation
- [ ] Methodology documented
- [ ] Code commented and organized
- [ ] Data sources cited
- [ ] Assumptions stated
- [ ] Reproducibility ensured

### Communication
- [ ] Executive summary prepared
- [ ] Visualizations created
- [ ] Key findings highlighted
- [ ] Limitations acknowledged
- [ ] Recommendations prioritized
